# スコアリング設計書

## 概要

診断は3フェーズ。各フェーズの回答からスコアを算出し、最終的に6軸0〜100に正規化する。

---

## 6軸の定義

| 軸 (key) | 日本語 | 意味 |
|---|---|---|
| dominance | 支配 | 相手の行動・環境をコントロールすることへの欲求 |
| submission | 服従 | 決定権を他者に委ね、責任から解放される欲求 |
| sadism | 加虐 | 相手の感情の揺れや反応を引き出すことへの快感 |
| masochism | 被虐 | 痛みや冷たさを愛情の証として受け入れる欲求 |
| psychological | 心理 | 視線・沈黙・心理戦など精神的な作用への嗜好 |
| sensory | 五感 | 音・温度・質感・気配など五感による没入への嗜好 |

※ `bondage`, `discipline`, `exhibitionism`, `voyeurism` は内部軸として存在するが、最終的にGeminiへ渡す6軸には含まれない（正規化後に除外）。

---

## スコア計算の仕組み

### Step 1: 生スコア加算

```
Phase1: 各質問に「はい」→ 指定軸に +weight (weight=1)
Phase2: 各シナリオの選択肢 → 指定軸に +weight (weight=2)
```

プラスのみの加算。マイナスは現状未実装。

### Step 2: 正規化

```
max = 全軸の最大値
各軸 = round(生スコア / max * 100)
```

最も高い軸が100になるよう全軸を引き伸ばす相対スコア。

## マイナス追加のための注意事項

### 現行の正規化の問題

正規化は「全軸の最大値を100とする相対スコア」。  
マイナスを追加すると生スコアが0未満になりうる。  
その場合、正規化後も0未満（例: -20）が出る。

対処案として別AIに依頼する場合：

- **案A**: 正規化前にすべての軸に`offset`を加えて下限を0にクランプ  
  例: `raw[axis] = Math.max(0, raw[axis])`
- **案B**: 正規化後に`Math.max(0, score)`でクランプ（現状の`normalize()`に追記するだけ）
- **案C**: 生スコアの最小値をshiftして全体を正の領域に移動してから正規化

### スコア設計の依頼フォーマット例

別AIへの依頼時にこのような形式で返してもらうと実装しやすい：

```
Q1: dominance +1, submission -1
Q16: submission +1, dominance -1
...
```

Phase2も同様：

```
S1-A: submission +3, dominance -2
```
