# プロジェクト要件定義書：Bain de Secret Miroir（診断特化型）

## 1. プロジェクト概要

### サービス名
**Bain de Secret Miroir**（バン・ド・スクレ・ミロワール）

### コンセプト
AIによる深層心理分析を用いた「大人のための性癖自己発見コンテンツ」。

### ビジネスモデル
- **完全無料・広告収益モデル**
- マッチング機能を排し、匿名で楽しめるエンタメ診断に特化
- 法的リスクと運営コストを最小化

### デザインテーマ
- 漆黒（ジェットブラック）
- ディープブルー（深海のような青）
- モダン・クール・エレガント

## 2. ユーザーフロー（画面遷移）

没入感を高めつつ、自然に広告を提示するフローを設計。

### 2.1 Top / Start
- コンセプトを伝える美麗なランディングページ
- ボタン一つで即診断開始
- ユーザー登録・ログイン不要

### 2.2 Phase 1: Swipe（直感スワイプ）
- **問題数**: 50問
- **UI**: カードが左右に流れる滑らかなアニメーション
- **目的**: 直感的な反応を収集

### 2.3 Phase 2: Situation（シチュエーション選択）
- **問題数**: 6問
- **UI**: 物語を読み進めるノベルゲーム形式
- **目的**: 具体的なシチュエーションでの傾向を把握

### 2.4 Phase 3: Text（自由記述）
- **問題数**: 2問
- **UI**: 自分と向き合う静かな入力時間
- **目的**: 深層心理の言語化

### 2.5 Analyzing（広告表示タイミング）
- **演出**: 「AIが深淵を解析中…」のローディング画面
- **広告**: フルスクリーン広告（インタースティシャル広告）
- **時間**: 5〜10秒程度

### 2.6 Result（診断結果）
- **診断結果カード**の提示
- **SNSシェアボタン**を配置
- **カード下部にネイティブ広告**を配置

## 3. 診断ロジック

AI（Gemini API）が「定量（スコア）」と「定性（文章）」を組み合わせて最終判定を行う。

### 3.1 スコアリング
BDSMの4軸 + 五感の反応を数値化：
- **D/S軸**: Dominant（支配）/ Submissive（服従）
- **B/D軸**: Bondage（拘束）/ Discipline（調教）
- **S/M軸**: Sadism（加虐）/ Masochism（被虐）
- **心理的背徳軸**: 禁忌への憧れ度合い
- **五感反応**: 視覚・聴覚・触覚・嗅覚・味覚への敏感度

### 3.2 テキスト解析
自由記述から以下を抽出：
- ユーザーの言葉のトーン
- 具体的な欲望の対象
- 潜在的な傾向

### 3.3 AIプロンプト設計
- **System**: エレガントな心理アナリストの役割を定義
- **Output**: システム処理用のJSONを強制
  - タイプ名
  - スコア配列
  - 診断文章
  - 相性の良いパートナータイプ

## 4. 拡散・収益化戦略

「バズ」を最大化し、トラフィックを広告収益に変える。

### 4.1 SNSシェア特化
- **OGP画像の動的生成**
  - 診断結果のタイプ名を美しく印字
  - ブランドカラー（漆黒×ディープブルー）を使用
- **シェアテキスト**
  - 「あなたに合うパートナーのタイプ」を提示
  - 知人間の話題性を誘発

### 4.2 広告配置
1. **診断待機中**
   - インタースティシャル広告（全画面）
   - ユーザーは「解析中」と認識し、抵抗感が少ない

2. **結果画面**
   - インフィード広告（デザインに馴染む形式）
   - ネイティブ広告で違和感を最小化

### 4.3 広告プラットフォーム候補
- Google AdSense
- Taboola / Outbrain（ネイティブ広告）
- A8.net（アフィリエイト）

## 5. 技術スタック（維持費0円・セキュア構成）

### 5.1 フロントエンド
- **技術**: React (SPA)
- **ホスティング**: Vercel または Cloudflare Pages
- **特徴**: アカウント管理不要の「使い切り型」サービス
- **メリット**: 静的サイトとして配信可能、サーバーレス、無料枠が大きい

### 5.2 バックエンド
- **技術**: AWS Lambda + API Gateway
- **設計**: ステートレス
- **機能**: DB保存を最小限にし、APIを中継するのみ
- **コスト**: 月間100万リクエストまで無料

### 5.3 AI API
- **サービス**: Google Gemini API (1.5 Pro / Flash)
- **プラン**: 有料枠（従量課金）
- **理由**: 
  - 「学習利用禁止」を徹底
  - ユーザーのプライバシーを保護
  - APIキーによる課金管理が可能

### 5.4 データベース（オプション）
- 基本的にはセッションストレージのみ使用
- 必要に応じて以下を検討：
  - **Supabase**（PostgreSQL、無料枠あり）
  - **Firebase Firestore**（NoSQL、無料枠あり）
- **用途**: 統計情報の収集（匿名）

## 6. セキュリティ・プライバシー設計

### 6.1 データ保存ポリシー
- **ユーザーの回答内容を永続的に保存しない**
- セッション終了後はすべて削除
- 統計情報のみ匿名化して収集

### 6.2 API通信
- HTTPS必須
- APIキーはサーバーサイドで管理
- CORS設定でドメイン制限

### 6.3 コンプライアンス
- 利用規約・プライバシーポリシーの明記
- GDPR / 個人情報保護法への対応
- 18歳以上の確認画面（年齢認証）

## 8. KPI（重要指標）

### 8.1 ユーザー関連
- **PV数**: ページビュー
- **診断完了率**: 開始→完了の割合
- **平均滞在時間**: ユーザーの没入度

### 8.2 拡散関連
- **シェア数**: Twitter, Instagram, Facebookでのシェア
- **OGP経由流入**: SNS経由の新規訪問
- **リピート率**: 再訪問率

### 8.3 収益関連
- **広告表示回数**: インプレッション
- **クリック率**: CTR
- **収益額**: RPM（1000PVあたりの収益）
